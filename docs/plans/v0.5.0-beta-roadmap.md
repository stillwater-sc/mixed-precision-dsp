# v0.5.0 Beta Roadmap

Target: close the most visible algorithmic gaps, broaden number type coverage
for real-time signal processing research, and make the mixed-precision value
proposition immediately demonstrable.

## Tier 1 — Core algorithmic gaps

### 1. Zero-phase filtering (`filtfilt`)

Forward-backward biquad cascade filtering with transient handling.
Classic DSP operation; straightforward to implement on the existing
`Cascade` infrastructure. Enables fair comparison with KFR and MATLAB.

**Files:** `include/sw/dsp/filter/filtfilt.hpp`
**Effort:** Small

### 2. Parks-McClellan (Remez exchange) FIR design

The README promises "equiripple FIR" and the KFR comparison lists it.
Without it, FIR design is limited to window methods only. The Remez
exchange algorithm produces optimal equiripple FIR filters for given
passband/stopband specs.

**Files:** `include/sw/dsp/filter/fir/remez.hpp`
**Effort:** Medium — the algorithm is well-documented but numerically
sensitive; needs careful implementation in the three-scalar model.

### 3. Bluestein / chirp-z FFT for arbitrary lengths

Radix-2-only is a hard limitation. Bluestein's algorithm (via chirp-z
transform and convolution with a known-length FFT) unlocks arbitrary-length
DFTs. This unblocks PSD and spectrogram on real-world signal lengths
without wasteful zero-padding.

**Files:** `include/sw/dsp/spectral/bluestein.hpp`, update `fft.hpp`
**Effort:** Medium

## Tier 2 — Strengthen the mixed-precision story

### 4. Expanded number type coverage

The library currently uses a narrow slice of Universal's type system.
For credible mixed-precision DSP research, we need systematic coverage
of the formats relevant to real-time signal processing.

See [mixed-precision-format-design.md](../mixed-precision-format-design.md)
for the full rationale, historical context, and type matrix.

#### 4a. Standard posits (es=2): p8, p16, p32 + real-time sizes p6, p12, p24

Standard posits use es=2 throughout, enabling trivial up/down conversion
across the pipeline:

| Name | Type | Role |
|------|------|------|
| p32  | `posit<32,2>` | Coefficient design, high-precision state |
| p24  | `posit<24,2>` | Balanced state accumulator |
| p16  | `posit<16,2>` | Standard sample streaming |
| p12  | `posit<12,2>` | Compact real-time samples |
| p8   | `posit<8,2>`  | Ultra-low-precision edge/sensor |
| p6   | `posit<6,2>`  | Minimum viable DSP exploration |

#### 4b. High-precision posits (es=0 and es=1)

For applications needing precision over dynamic range — an alternative
to fixed-point with tapered accuracy:

**es=0 (no exponent scaling — maximum precision near 1.0):**
`posit<4,0>`, `posit<6,0>`, `posit<8,0>`, `posit<12,0>`, `posit<16,0>`

**es=1 (moderate dynamic range with good precision):**
`posit<4,1>`, `posit<6,1>`, `posit<8,1>`, `posit<12,1>`, `posit<16,1>`

#### 4c. Fixed-point formats informed by historical DSP practice

Classical DSP processors used specific fixed-point formats for specific
pipeline stages. We want to bring back this domain knowledge:

| Format | Q-notation | Use case |
|--------|-----------|----------|
| `fixpnt<8,7>`   | Q1.7  | 8-bit sample I/O, normalized [-1,1) |
| `fixpnt<12,11>`  | Q1.11 | 12-bit ADC samples (radar, sonar) |
| `fixpnt<16,15>`  | Q1.15 | Standard 16-bit audio / telephony |
| `fixpnt<16,14>`  | Q2.14 | Filter coefficients with headroom |
| `fixpnt<24,23>`  | Q1.23 | Professional audio (24-bit ADC) |
| `fixpnt<32,31>`  | Q1.31 | Extended accumulator |
| `fixpnt<32,16>`  | Q16.16 | Wide dynamic range accumulator |
| `fixpnt<40,31>`  | Q9.31 | Guard-bit accumulator (16x16 MAC + 8 guard) |
| `fixpnt<48,32>`  | Q16.32 | Double-width accumulator for 24-bit paths |

#### 4d. Logarithmic Number System (LNS)

LNS represents values as fixed-point logarithms: multiplication becomes
addition, which is attractive for filter coefficient application.
Currently unused in the library.

| Format | Notes |
|--------|-------|
| `lns<8,5>`   | 8-bit, 5 fractional bits in log domain |
| `lns<12,8>`  | Compact multiplier-free DSP |
| `lns<16,10>` | Standard LNS word |
| `lns<32,22>` | High-precision LNS |

#### 4e. Custom floats (cfloat)

Extend existing coverage to include formats relevant to ML/DSP crossover:

| Format | Analogue | Notes |
|--------|----------|-------|
| `cfloat<8,2>`  | E2M5 | Narrow ML inference |
| `cfloat<8,4>`  | E4M3 (FP8) | ML training/inference |
| `cfloat<8,5>`  | E5M2 (FP8) | ML gradient format |
| `cfloat<16,5>` | IEEE binary16 | Standard half |
| `cfloat<16,8>` | bfloat16 | ML accelerator format |
| `cfloat<24,5>` | — | Extended half precision |
| `cfloat<32,8>` | IEEE binary32 | Standard float |

#### 4f. Integer types

For quantization studies and ADC/DAC modeling:

`integer<6>`, `integer<8>`, `integer<10>`, `integer<12>`,
`integer<14>`, `integer<16>`, `integer<24>`

### 5. Higher-order noise shaping

Only first-order error-feedback noise shaping exists. Second-order and
third-order noise shapers are what sigma-delta ADCs actually use and
are directly relevant to the precision argument. Add at minimum
second-order and third-order variants.

**Files:** `include/sw/dsp/quantization/noise_shaping.hpp`
**Effort:** Small

### 6. Rational sample-rate conversion

Polyphase up/down works for integer factors. Arbitrary L/M ratio
conversion (with anti-alias filtering) is needed for real audio and
SDR pipelines.

**Files:** `include/sw/dsp/filter/fir/polyphase.hpp` or new
`include/sw/dsp/conditioning/src.hpp`
**Effort:** Medium

### 7. Precision sweep CLI tool

Standalone application that takes a filter specification (family, order,
cutoff, sample rate) and sweeps across a configurable set of Universal
number types. Produces a summary table of: SQNR, max pole displacement,
stability margin, coefficient condition number, and passband/stopband
error. Makes the library's value proposition immediately demonstrable.

**Files:** `applications/precision_sweep/`
**Effort:** Medium — builds on existing analysis infrastructure

## Tier 3 — Polish

### 8. Additional window functions

Currently 6 windows (Hamming, Hanning, Blackman, Kaiser, rectangular,
flat-top). Add Tukey (cosine-tapered), Gaussian, Dolph-Chebyshev, and
Bartlett-Hann to close the gap with KFR's 14+.

**Files:** `include/sw/dsp/windows/`
**Effort:** Small
