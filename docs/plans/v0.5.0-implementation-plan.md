## 1. Zero-phase filtering (`filtfilt`)

### Design

`filtfilt` applies a filter forward and then backward, cancelling phase
distortion. The standard algorithm (MATLAB convention):

1. Extend input with reflected samples to reduce transients
2. Filter forward through the cascade → intermediate signal
3. Reverse the intermediate signal
4. Filter forward again (= backward pass on original)
5. Reverse the result

The reflection length is typically `3 * max(len(a), len(b)) - 1` for
IIR filters, which translates to `3 * (2 * num_stages + 1) - 1` for
a biquad cascade. Initial conditions are computed by solving the
filter's steady-state equation for the reflected signal edge value.

### API

```cpp
// include/sw/dsp/filter/filtfilt.hpp

template <DspField CoeffScalar, int MaxStages,
          typename StateForm = DirectFormII<CoeffScalar>,
          DspScalar SampleScalar = CoeffScalar>
mtl::vec::dense_vector<SampleScalar> filtfilt(
    const Cascade<CoeffScalar, MaxStages>& cascade,
    const mtl::vec::dense_vector<SampleScalar>& input);
```

### Implementation steps

1. Compute reflection length from `cascade.num_stages()`
2. Extend input: `[2*x[0]-x[nrefl], ..., 2*x[0]-x[1], x[0], ..., x[N-1], 2*x[N-1]-x[N-2], ..., 2*x[N-1]-x[N-1-nrefl]]`
3. Forward pass: `cascade.process()` through all extended samples
4. Reverse the filtered output in-place
5. Reset state, forward pass again
6. Reverse again, extract the central N samples

### Test plan

- Verify output has zero phase shift (symmetric impulse response)
- Verify magnitude response is squared vs single-pass
- Compare against known reference for a 4th-order Butterworth
- Test with different state forms (DFI, DFII, TDFII)
- Test with posit and fixpnt sample types

- `include/sw/dsp/filter/filtfilt.hpp`
- `tests/test_filtfilt.cpp`

## 2. Parks-McClellan (Remez exchange) FIR design

### Design

The Remez exchange algorithm iteratively finds the optimal equiripple
FIR filter by solving the Chebyshev approximation problem on alternating
extremal frequencies. The algorithm:

1. Initialize extremal frequency set (N+2 points for N+1 taps)
2. Solve Lagrange interpolation to find the optimal deviation δ
3. Evaluate the error function on a dense grid
4. Update extremal set to the N+2 largest error peaks
5. Repeat until convergence (δ change < tolerance)

### API

```cpp
// include/sw/dsp/filter/fir/remez.hpp

enum class RemezBandType { bandpass, differentiator, hilbert };

template <DspField T>
mtl::vec::dense_vector<T> remez(
    std::size_t num_taps,
    const std::vector<T>& bands,     // [f1,f2, f3,f4, ...] normalized [0,0.5]
    const std::vector<T>& desired,   // [d1,d2, d3,d4, ...] gain per band
    const std::vector<T>& weights,   // [w1, w2, ...] weight per band
    RemezBandType type = RemezBandType::bandpass,
    int max_iterations = 40,
    int grid_density = 16);
```

Convenience wrappers:

```cpp
template <DspField T>
mtl::vec::dense_vector<T> design_fir_equiripple_lowpass(
    std::size_t num_taps, T passband_edge, T stopband_edge,
    T passband_weight = T{1}, T stopband_weight = T{1});

template <DspField T>
mtl::vec::dense_vector<T> design_fir_equiripple_bandpass(
    std::size_t num_taps, T stop1, T pass1, T pass2, T stop2,
    T stopband_weight = T{1}, T passband_weight = T{1});
```

### Implementation steps

1. Build the dense frequency grid (grid_density × num_taps points per band)
2. Initialize extremal set with uniform spacing
3. Lagrange interpolation loop:
   a. Compute deviation δ from the current extremal set
   b. Evaluate ideal response + weighted deviation across all grid points
   c. Find all local maxima of the error
   d. Select new extremal set (N+2 largest error extrema with alternating sign)
4. Final Lagrange interpolation → frequency response → IDFT → tap coefficients
5. Convergence check: relative change in δ < 1e-12

### Key numerical considerations

- Barycentric Lagrange interpolation for numerical stability
- The algorithm must be implemented in `double` internally (or `CoeffScalar`
  if it satisfies `ConvertibleToDouble`) — the Remez exchange is inherently
  a high-precision design-time computation
- Output taps can be projected to any `DspField` type

### Test plan

- Compare 31-tap equiripple lowpass against MATLAB/scipy `remez()` reference
- Verify equiripple property: all extrema in passband and stopband have
  equal magnitude deviation
- Verify transition band width matches specification
- Test differentiator and Hilbert transformer modes
- Test with odd and even tap counts (Type I and Type II linear phase)

### Files

- `include/sw/dsp/filter/fir/remez.hpp`
- `tests/test_remez.cpp`
- `CMakeLists.txt` (register test)

## 3. Bluestein FFT for arbitrary lengths

### Design

Bluestein's algorithm converts an N-point DFT into a circular convolution
of length M ≥ 2N-1 (rounded up to next power-of-2 for radix-2 FFT).
This enables arbitrary-length DFTs using the existing radix-2 FFT:

1. Precompute chirp sequence: `w[n] = exp(-jπn²/N)` for n = 0..N-1
2. Multiply input by chirp: `a[n] = x[n] * w[n]`
3. Build chirp convolution kernel: `b[n] = conj(w[n])`, zero-padded to M
4. Convolve a and b using radix-2 FFT: `c = IFFT(FFT(a_padded) * FFT(b))`
5. Multiply result by chirp: `X[k] = w[k] * c[k]` for k = 0..N-1

### API

```cpp
// include/sw/dsp/spectral/bluestein.hpp

template <DspField T>
void bluestein_forward(mtl::vec::dense_vector<complex_for_t<T>>& data);

template <DspField T>
void bluestein_inverse(mtl::vec::dense_vector<complex_for_t<T>>& data);
```

Update `fft.hpp` to auto-dispatch:

```cpp
template <DspField T>
void dft_forward(mtl::vec::dense_vector<complex_for_t<T>>& data) {
    if (is_power_of_2(data.size()))
        fft_forward<T>(data);    // radix-2 fast path
    else
        bluestein_forward<T>(data);  // arbitrary length
}
```

### Test plan

- Verify against naive DFT for prime lengths (7, 13, 127, 251)
- Verify against radix-2 FFT for power-of-2 lengths (should match)
- Test Parseval's theorem (energy preservation)
- Test with posit and cfloat types
- Benchmark overhead vs zero-padded radix-2

### Files

- `include/sw/dsp/spectral/bluestein.hpp`
- Update `include/sw/dsp/spectral/fft.hpp` (add `dft_forward`/`dft_inverse`)

## 4. Expanded number type coverage (precision sweep)

### Design

Expand the existing `iir_precision_sweep` application into a
comprehensive precision sweep tool that covers all five number systems.
The tool should be the flagship demonstration of the library's value.

### Type matrix

The sweep should cover these configurations in a three-scalar grid:

**Coefficient types:** `double`, `posit<32,2>`, `posit<16,0>`,
`fixpnt<32,24>`, `fixpnt<16,14>`, `lns<32,22>`, `cfloat<32,8>`

**State types:** `double`, `posit<32,2>`, `posit<24,2>`, `posit<16,2>`,
`fixpnt<32,31>`, `fixpnt<40,31>`, `lns<16,10>`, `cfloat<24,5>`

**Sample types:** `float`, `posit<16,2>`, `posit<12,2>`, `posit<8,2>`,
`posit<16,1>`, `posit<8,1>`, `posit<16,0>`, `posit<8,0>`,
`fixpnt<16,15>`, `fixpnt<12,11>`, `fixpnt<8,7>`,
`cfloat<16,5>`, `cfloat<8,4>`, `lns<16,10>`, `lns<8,5>`,
`integer<16>`, `integer<12>`, `integer<8>`

Not a full Cartesian product — the sweep should test meaningful
pipeline configurations:

1. **Uniform precision:** same type at all three positions
2. **Classic mixed:** `double` coefficients, varied state, varied samples
3. **Posit pipeline:** `posit<32,2>` → `posit<24,2>` → `posit<N,2>`
4. **Fixed-point pipeline:** `fixpnt<32,24>` → `fixpnt<32,31>` → `fixpnt<16,15>`
5. **Cross-system:** `double` coeff → `posit<32,2>` state → `fixpnt<16,15>` samples
6. **LNS experiment:** `double` coeff → `lns<16,10>` state → various samples

### Measurements per configuration

- SQNR (dB) vs double reference
- Max coefficient quantization error
- Pole displacement (L2 norm)
- Stability margin (distance of worst pole to unit circle)
- Passband ripple (dB)
- Stopband attenuation (dB)
- Condition number of the biquad cascade

### Output

- Console summary table
- CSV for Python/MATLAB visualization
- Per-configuration frequency response CSV

### Files

- `applications/precision_sweep/main.cpp`
- `applications/precision_sweep/CMakeLists.txt`
- Update root `CMakeLists.txt` to include subdirectory

## 5. Higher-order noise shaping

### Design

Extend `FirstOrderNoiseShaper` with second-order and third-order
variants. The error feedback filter becomes:

- **Second-order:** `feedback = -2*e[n-1] + e[n-2]` (second-difference)
- **Third-order:** `feedback = -3*e[n-1] + 3*e[n-2] - e[n-3]`

These correspond to (1 - z⁻¹)² and (1 - z⁻¹)³ noise transfer functions.

### API

```cpp
template <DspField HighPrecT, DspField LowPrecT>
class SecondOrderNoiseShaper {
    LowPrecT process(HighPrecT input);
    void reset();
};

template <DspField HighPrecT, DspField LowPrecT>
class ThirdOrderNoiseShaper {
    LowPrecT process(HighPrecT input);
    void reset();
};
```

### Test plan

- Verify noise spectral tilt: 2nd order should show 40 dB/decade,
  3rd order 60 dB/decade high-frequency noise emphasis
- Compare in-band SQNR improvement over unshaped quantization
- Test stability (higher-order shapers can be unstable with aggressive
  quantization — verify with small word widths)

### Files

- Update `include/sw/dsp/quantization/noise_shaping.hpp`
- `tests/test_noise_shaping.cpp`

## 6. Rational sample-rate conversion

### Design

Combine polyphase interpolation (factor L) and decimation (factor M)
with an anti-alias filter designed for the higher rate. For L/M
conversion:

1. Upsample by L (insert L-1 zeros between samples)
2. Filter with lowpass at min(π/L, π/M)
3. Downsample by M (take every M-th sample)

The polyphase structure avoids computing samples that will be discarded.

### API

```cpp
template <DspField CoeffScalar = double,
          DspField StateScalar = CoeffScalar,
          DspScalar SampleScalar = StateScalar>
class RationalResampler {
    RationalResampler(std::size_t L, std::size_t M,
                      std::size_t filter_half_length = 10,
                      CoeffScalar beta = CoeffScalar{5});

    mtl::vec::dense_vector<SampleScalar> process(
        const mtl::vec::dense_vector<SampleScalar>& input);

    void reset();
    double ratio() const { return double(L_) / double(M_); }
};
```

The constructor auto-designs a Kaiser-windowed sinc anti-alias filter
of length `2 * filter_half_length * max(L, M) + 1`.

### Test plan

- 44100 → 48000 Hz (L=160, M=147): verify frequency content preservation
- 48000 → 16000 Hz (L=1, M=3): verify anti-alias filtering
- Roundtrip: resample up then down, measure SNR vs original
- Test with posit and fixpnt sample types

### Files

- `include/sw/dsp/conditioning/src.hpp`
- `tests/test_src.cpp`
- `CMakeLists.txt`

## 7. Additional window functions

### Design

Add four windows to bring the total from 6 to 10:

**Tukey (cosine-tapered):**
Parameterized by α ∈ [0,1] where α=0 is rectangular, α=1 is Hanning.

**Gaussian:**
Parameterized by σ: `w[n] = exp(-0.5 * ((n - N/2) / (σ * N/2))²)`

**Dolph-Chebyshev:**
Equiripple sidelobes at specified attenuation level. Uses inverse DFT
of Chebyshev polynomial evaluated on the unit circle.

**Bartlett-Hann:**
`w[n] = 0.62 - 0.48|n/(N-1) - 0.5| + 0.38*cos(2π(n/(N-1) - 0.5))`

### API

```cpp
template <DspField T>
mtl::vec::dense_vector<T> tukey_window(std::size_t length, double alpha = 0.5);

template <DspField T>
mtl::vec::dense_vector<T> gaussian_window(std::size_t length, double sigma = 0.4);

template <DspField T>
mtl::vec::dense_vector<T> dolph_chebyshev_window(std::size_t length, double attenuation_db = 100.0);

template <DspField T>
mtl::vec::dense_vector<T> bartlett_hann_window(std::size_t length);
```
